How this site maintains itself

Nobody edits this site by hand. It researches, hosts, and updates itself, and this page explains the loop.

The loop

  1. Parallel's research API filled the comparison table once, with cited sources for every field.
  2. Parallel web monitors then watch for change: one per provider re-checks specs and pricing daily, and another looks for new sandbox products.
  3. The whole site runs in a single Sailbox, a cloud VM that checkpoints itself after about a minute of quiet. While it sleeps, it costs nothing.
  4. When a monitor finds something, its webhook lands here and wakes the VM in a few seconds. The same frozen process picks the request up.
  5. An opencode agent runs inside the VM, edits the data the event supports, and commits. One wake becomes one commit and one entry on the update log.

Visits do the same thing. If this page took a moment to load, you woke the server up.

What it costs

Sleeping is free; the bill only runs while the VM does. A typical update keeps it awake for a few minutes, so hosting plus updates lands around a dollar a month. The log page shows the estimate per update.

Where the data comes from

Public sources only: official docs, pricing pages, and changelogs, all linked from each product page. This site is built on two of the products it covers. Sailboxes is a row in the table and also the host, and Parallel supplies the research. Read those rows with the same skepticism as any other. The sources are linked and the git history records every change.
